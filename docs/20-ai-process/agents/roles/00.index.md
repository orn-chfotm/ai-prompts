# roles 인덱스

이 디렉토리는 역할별 상세 규칙을 담는다.

각 역할 문서는 다음 관점을 포함한다.

- 역할 인물상
- 핵심 역량
- 판단 기준
- 책임
- 해야 할 질문
- 산출물
- 제한

## 파일 목록

- `01-pm.md`: 사용자의 요청을 업무 구조와 승인 지점으로 정리하는 PM
- `02-cto.md`: 폭넓은 개발 지식으로 기술 방향과 리스크를 검토하는 CTO
- `03-pl.md`: CTO 방향을 개발 계획과 PA 작업 단위로 바꾸는 PL
- `04-pa.md`: PL이 분리한 작업을 정확하게 실행하는 PA

## 기본 흐름

1. PM이 업무 구조를 만든다.
2. CTO가 기술 방향과 제한성을 검토한다.
3. PL이 개발 작업을 분리한다.
4. PA가 승인된 작업 단위를 실행한다.
5. 사용자가 승인, 거부, 보류, 재요청을 결정한다.

## 역할 사용 기준

작은 문서 수정이나 단순 확인 작업은 PA 중심으로 처리할 수 있다.

설계, 아키텍처, 의존성, 보안, 배포, 다중 작업이 포함되면 PM, CTO, PL 검토를 생략하지 않는다.
