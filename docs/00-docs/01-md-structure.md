# md 구조 규칙

이 문서는 `docs` 하위 md 파일의 작성 기준을 정의한다.

## 최상위 구조

최상위 `docs`에는 md 파일을 직접 두지 않는다.

`docs` 하위에는 대분류 디렉토리만 둔다.

```text
docs/
  00-docs/
  10-technical/
  20-ai-process/
  30-execution/
  40-plan/
  50-review/
```

## 번호 규칙

- 대분류 디렉토리는 10단위 번호를 사용한다.
- 대분류 하위 디렉토리는 번호를 붙이지 않고 주제명만 사용한다.
- 각 디렉토리 안의 md 파일은 `00.index.md`, `01-*`, `02-*` 순서로 작성한다.
- `00.index.md`는 해당 디렉토리의 목적, 파일 목록, 읽는 순서를 설명한다.

## front matter

각 md 파일 상단에는 `description`을 둔다.

```md
---
description: "문서의 목적을 한 문장으로 설명합니다."
---
```

## Agent 진입점

Codex와 Claude는 공식 로딩 방식이 다르다.

- Codex는 `AGENTS.md`에서 참조 문서 목록을 안내한다.
- Claude는 `CLAUDE.md` 상단에서 `@docs/...` import를 사용한다.
- 공통 문서에는 특정 Agent만 이해하는 문법을 넣지 않는다.
- `AGENTS.md`와 `CLAUDE.md`는 docs 하네스를 연결하는 adapter로 취급한다.
- Agent별 실제 실행 커스터마이징은 `.codex`와 `.claude`에 두되, 공통 하네스 문서와 섞지 않는다.
- Codex skill은 `.codex/skills`가 아니라 `.agents/skills`에 둔다.
- Agent별 기본 디렉토리 생성 기준은 `03-agent-dir-init.md`를 따른다.

## 문서 이동 규칙

문서를 이동하면 다음 파일의 참조를 함께 갱신한다.

- `AGENTS.md`, 해당 프로젝트에서 Codex adapter로 사용하는 경우
- `CLAUDE.md`, 해당 프로젝트에서 Claude adapter로 사용하는 경우
- 관련 디렉토리의 `00.index.md`
- 문서 내부의 예시 경로

## 파일 탐색 원칙

Agent는 매 작업마다 전체 repository를 풀 스캔하지 않는다.

이 원칙은 이 하네스를 가져가 사용하는 실제 프로젝트 소스에 우선 적용한다.

하네스 문서 자체를 유지보수하는 경우에는 변경 대상 `docs` 문서와 직접 참조되는 주변 문서를 중심으로 읽는다.

문서나 파일을 읽을 때는 다음 순서를 따른다.

1. 사용자가 직접 언급한 파일을 먼저 읽는다.
2. 해당 파일이 참조하는 가까운 문서를 읽는다.
3. 필요한 경우 관련 디렉토리의 `00.index.md`를 읽는다.
4. 그래도 부족할 때만 제한된 검색을 수행한다.

전체 스캔은 다음 경우에만 허용한다.

- 사용자가 전체 구조 검토를 요청한 경우
- 참조 경로가 깨져 관련 파일을 찾을 수 없는 경우
- 리팩터링, 문서 재구조화처럼 전체 영향 범위 확인이 필요한 경우
- 실제 프로젝트 소스의 영향 범위를 제한 검색으로 판단하기 어려운 경우

전체 스캔이 필요하면 목적과 범위를 먼저 설명한다.

## commit 기준

일반적으로 commit 대상은 `docs` 하위 공통 하네스 문서다.

`.codex`와 `.claude`는 다음 경우에만 commit 대상으로 고려한다.

- 모든 프로젝트에서 동일하게 사용할 Agent adapter가 필요하다.
- 팀 공통 skill, rule, agent 정의로 합의되었다.
- 로컬 개인 설정, 프로젝트별 임시 설정, 실험용 설정이 아니다.
