# 오류 처리

오류는 발견 즉시 숨기지 않고 보고한다.

## 오류 분류

- 실행 오류: 명령, 빌드, 테스트 실행 중 발생
- 구현 오류: 요구사항과 다른 동작
- 통합 오류: PA 작업 간 충돌
- 설계 오류: CTO 방향 또는 PL 계획과 충돌
- 승인 오류: 사용자 승인 범위를 벗어난 작업

## 처리 기준

- 실행 오류는 PA가 원인을 정리해 PL에게 보고한다.
- 통합 오류는 PL이 작업 경계를 재조정한다.
- 설계 오류는 CTO 검토로 되돌린다.
- 승인 오류는 사용자 승인 요청으로 되돌린다.
- 재현 가능한 오류와 크리티컬 문제는 `04-review-logging.md` 기준에 따라 실제 프로젝트에 기록한다.

## 재작업 기준

- 범위 오류: PM 단계
- 기술 방향 오류: CTO 단계
- 작업 분리 오류: PL 단계
- 구현 오류: PA 단계
- 검증 오류: review 단계

## 기록 기준

다음 오류는 반드시 로그로 남긴다.

- 같은 원인으로 반복될 수 있는 오류
- 테스트 실패
- 빌드 실패
- 보안 또는 권한 관련 오류
- 데이터 손상 가능성이 있는 오류
- 사용자 승인 범위를 벗어난 변경
