# 50-review 인덱스

이 디렉토리는 작업 완료 후 어떤 방식으로 review하고, 오류와 test를 어떻게 처리할지 정의한다.

## 파일 목록

- `01-review-process.md`: 리뷰 절차와 역할별 검토 기준
- `02-test-policy.md`: 테스트 실행과 검증 보고 기준
- `03-error-handling.md`: 오류 발견, 보고, 재작업 기준
- `04-review-logging.md`: 리뷰 중 발견한 에러, 크리티컬, 개선 사항과 AI 설계/피드백/개선점 기록을 실제 프로젝트에 남기는 기준

## 책임

- 작업 결과의 품질을 확인한다.
- 실패한 test와 오류를 숨기지 않는다.
- 재작업이 필요한 경우 어느 단계로 되돌릴지 정한다.
- 사용자에게 최종 승인 가능한 형태로 결과를 보고한다.
- submodule로 가져간 하네스 내부가 아니라, 하네스를 사용하는 실제 프로젝트 구조 안에 리뷰 기록을 남긴다.
