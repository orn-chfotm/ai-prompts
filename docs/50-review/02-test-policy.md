# 테스트 정책

테스트는 작업의 신뢰도를 확인하기 위한 필수 검증 수단이다.

## 기본 원칙

- 가능한 테스트는 실행한다.
- 실행하지 못한 테스트는 이유를 남긴다.
- 실패한 테스트는 숨기지 않는다.
- 실패 원인이 작업 범위 밖이면 PL 또는 CTO에게 에스컬레이션한다.

## 보고 형식

```md
## 테스트 결과

- 실행한 테스트:
- 결과:
- 실패 항목:
- 실패 원인:
- 미실행 테스트:
- 미실행 사유:
```
