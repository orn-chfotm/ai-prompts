# Naming 규칙

이 문서는 프로젝트 전반의 이름 규칙을 관리한다.

## 기본 원칙

- 이름은 역할과 책임을 드러내야 한다.
- 약어는 팀에서 이미 공유된 경우에만 사용한다.
- 파일명, 클래스명, 함수명은 기존 프로젝트 컨벤션을 우선한다.
- 새 규칙이 필요하면 기존 패턴을 먼저 조사한 뒤 추가한다.

## Agent 적용 기준

PA는 naming을 임의로 바꾸지 않는다.

PL은 공통 naming 변경이 필요할 때 영향 범위를 정의한다.

CTO는 naming 변경이 API 계약, 데이터 스키마, 운영 규칙에 영향을 주는지 검토한다.
