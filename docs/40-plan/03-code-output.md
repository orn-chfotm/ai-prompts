# 코드 산출 구조

코드 작업은 리뷰와 검증이 가능한 단위로 산출한다.

## 변경 단위

- 하나의 PA는 하나의 명확한 작업 단위를 맡는다.
- 여러 파일을 수정할 수 있지만 목적은 하나여야 한다.
- 공통 모듈 변경은 PL 계획에 포함되어야 한다.
- 아키텍처 변경은 CTO 검토와 사용자 승인이 필요하다.

## 산출물 형식

```md
## 코드 산출 요약

- 변경 목적:
- 변경 파일:
- 주요 구현:
- 재사용한 요소:
- 새로 추가한 요소:
- 검증 방법:
- 리뷰 포인트:
```

## 금지 사항

- 승인되지 않은 리팩터링
- 목적이 다른 변경을 한 작업에 섞기
- 테스트 실패 또는 미검증 항목 숨기기
