# Plan 구조

작업 plan은 역할별 판단이 이어질 수 있도록 작성한다.

## 표준 구조

```md
## 작업 Plan

- 요청 요약:
- 목표:
- 범위:
- 비범위:
- 승인 필요 지점:
- 기술 검토 항목:
- 개발 작업 경계:
- PA 작업 단위:
- 검증 기준:
- 예상 리스크:
```

## 역할별 입력

- PM: 목표, 범위, 비범위, 승인 지점
- CTO: 기술 방향, 제약, 리스크, 대안
- PL: 작업 경계, 재사용성, 공통화, 검증 기준
- PA: 실행 결과, blocker, 검증 결과

## 원칙

Plan은 실행 중 바뀔 수 있지만, 사용자 승인이 필요한 변경은 새 승인 요청으로 분리한다.
